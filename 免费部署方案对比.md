# Render vs Koyeb vs 其他方案 — 缺点与风险完整分析

> 重点说缺点，不只看优点

---

## 一、Render 免费计划的缺点

### 痛点 1：15 分钟不访问就休眠（最烦人）

```
你 3 天没用 → 服务休眠
你再次打开网站 → Render 唤醒服务 → 等 30 秒才收到响应
唤醒后 15 分钟内不操作 → 又休眠
```

**对你意味着什么**：每次用工具前，先等 30 秒让服务醒过来。如果你每天用一次，就要等 30 秒一次。

### 痛点 2：服务器在美国

| 项目 | 问题 |
|---|---|
| **地理位置** | Render 只有美国 Oregon 和 Frankfurt（德国）节点 |
| **中国延迟** | 从中国访问 Render API 延迟约 200-300ms |
| **RDAP 查询** | 美国服务器查美国 RDAP 数据源（verisign 等）很快，< 50ms |
| **前端加载** | 前端页面从 Vercel 加载（Vercel 有香港节点），速度没问题 |
| **总体体验** | 前端快 + 后端慢半拍，每次操作感觉有点"顿" |

### 痛点 3：免费存储是临时的

Render 免费计划的磁盘是**临时的**——服务重启后所有文件消失：
- IANA Bootstrap 缓存 → 每次重启都要重新拉取（但可以内存缓存）
- SQLite 缓存 → **查过的域名下次没了**，没有缓存加速效果
- 你的查询历史？→ 没了

### 痛点 4：自定义域名要付费

免费计划只能用 `xxx.onrender.com` 的二级域名。想绑定自己的域名？**至少要 $7/月**。

### 痛点 5：免费额度刚好够用

| 项目 | 免费额度 | 你的消耗 |
|---|---|---|
| 运行时间 | 750 小时/月 | 如果 24h 跑 ≈ 744 小时/月 → **刚好够** |
| 带宽 | 100GB/月 | 域名查询传输量很小，够用 |
| 团队协作 | ✅ 无限 | 没问题 |

**但**：如果你一个月有 1 天 24 小时都在查，那 750 小时就超了，服务会被停掉。

## 二、Koyeb 免费计划的缺点

### 痛点 1：不支持原生 Python，必须用 Docker

Render 可以直接部署 Python（检测到 `requirements.txt` 就行），但 Koyeb **必须用 Docker**：

```
你的 Python 代码 → 必须写 Dockerfile → 构建镜像 → 推到 Koyeb
```

这意味着你多了一步：
```dockerfile
FROM python:3.11-slim
WORKDIR /app
COPY . .
CMD ["uvicorn", "server.main:app", "--host", "0.0.0.0", "--port", "$PORT"]
```

不算复杂，但比 Render 多一步。

### 痛点 2：平台还不成熟

| 项目 | Koyeb |
|---|---|
| 成立时间 | 2021 年 | 
| 用户规模 | 较小 |
| 文档质量 | 英文，一般 |
| 社区支持 | 少（遇到问题 GitHub Issues 或 Discord） |
| 稳定性 | 偶有服务中断（比 Render 不稳定） |

### 痛点 3：亚洲节点只有新加坡

Koyeb 有亚洲节点（新加坡），但**在中国访问新加坡的延迟仍然不低**（约 100-150ms）。比 Render 的美国节点好一些，但还是有延迟感。

而且新加坡节点有时资源不足（免费计划可能被分配到负载高的区域）。

### 痛点 4：免费资源少

| 项目 | Koyeb 免费 |
|---|---|
| 并发请求 | 1 个 Web Service（不能开多个服务） |
| 存储 | 1GB（比 Render 的临时存储好，有持久化） |
| 带宽 | 100GB/月 |
| 自定义域名 | ✅ 免费计划可以使用（**比 Render 好**） |

### 痛点 5：冷启动问题

Koyeb 虽然不休眠（比 Render 好），但**初次部署/更新时冷启动较慢**（构建镜像需要 2-5 分钟）。每次代码更新都要经历：

```
push 代码 → Koyeb 检测到变更 → 重新构建 Docker 镜像（2-5 分钟）
         → 启动新容器（30s）→ 切换流量
```

Render 在这点上稍微快一些（构建环境更好）。

## 四、这些缺点对你的实际影响有多大

### 取决于你的使用频率

**如果你每天查一次域名**：
- **Render** → 每次等 30s 唤醒 → 忍一下也行，但每次都要等
- **Koyeb** → 不用等，随时打开就用

**如果你偶尔查（一周一次）**：
- **Render** → 每次多等 30s，查完就走 → 其实还行
- **Koyeb** → 随时能用，但部署维护稍麻烦

**如果你在国内访问**：
- 两者都不算快（150-300ms 延迟）
- Koyeb（新加坡）比 Render（美国）好一些
- 要最快？只能选国内服务器（阿里云/腾讯云）

**如果你看重稳定性**：
- Render > Koyeb（Render 更成熟）
- 但 Render 的休眠问题更烦人

**Oracle 永久免费 VPS** 或 **自购 ¥50/月国内服务器** 才是最终方案：
