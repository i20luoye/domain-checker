# 域名批量查询工具免费部署方案

## 结论

你要的是免费 API，所以部署时只使用：

- RDAP：免费公开协议，用于 `.com/.net/.org/.ai/.app/.dev/.io` 等后缀
- WHOIS：免费公开协议，用于 `.cn`

不使用 GoDaddy、Namecheap、Dynadot 等商业 API，也不需要任何 API Key。

## 推荐架构

不要把 `domain_checker.html` 直接当静态网页部署。

原因：

- 浏览器直接请求第三方 RDAP 会遇到 CORS。
- `.cn` 需要走 CNNIC WHOIS 43 端口，浏览器不能直接连 TCP。
- 服务端统一代理查询后，客户只访问你的域名，稳定性更好。

推荐：

```text
浏览器 -> 你的服务器 /api/check -> 免费 RDAP / 免费 WHOIS
```

## 服务器准备

以 Ubuntu 为例：

```bash
sudo apt update
sudo apt install -y python3 python3-venv nginx
cd /opt
sudo mkdir domain-checker
sudo chown "$USER":"$USER" domain-checker
```

把项目文件上传到 `/opt/domain-checker`，然后安装依赖：

```bash
cd /opt/domain-checker
python3 -m venv .venv
. .venv/bin/activate
pip install -r requirements.txt
```

## 启动测试

明确指定免费模式：

```bash
export DOMAIN_CHECKER_SOURCE=rdap
python domain_server.py --host 0.0.0.0 --port 8000
```

浏览器打开：

```text
http://服务器IP:8000/
```

健康检查：

```bash
curl http://127.0.0.1:8000/api/health
```

返回里的 `source` 应该是：

```json
"source": "rdap"
```

## systemd 常驻运行

创建 `/etc/systemd/system/domain-checker.service`：

```ini
[Unit]
Description=Domain Checker Free RDAP WHOIS
After=network.target

[Service]
WorkingDirectory=/opt/domain-checker
ExecStart=/opt/domain-checker/.venv/bin/python /opt/domain-checker/domain_server.py --host 127.0.0.1 --port 8000
Restart=always
Environment=DOMAIN_CHECKER_SOURCE=rdap

[Install]
WantedBy=multi-user.target
```

启动：

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now domain-checker
sudo systemctl status domain-checker
```

## Nginx 反向代理

示例 `/etc/nginx/sites-available/domain-checker`：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

启用：

```bash
sudo ln -s /etc/nginx/sites-available/domain-checker /etc/nginx/sites-enabled/domain-checker
sudo nginx -t
sudo systemctl reload nginx
```

## 免费方案的能力边界

免费 RDAP/WHOIS 可以做：

- 判断域名已注册还是可注册
- 批量枚举查询
- `.cn` 通过 WHOIS 查询
- 给出“可能溢价”的启发式提示

免费 RDAP/WHOIS 不能做：

- 返回真实注册价格
- 准确判断注册商溢价价格
- 保证每个注册局都无限量高并发

建议页面文案写成：

```text
本工具使用公开 RDAP/WHOIS 数据源进行可用性查询。
“可能溢价”为算法提示，最终注册价格请以下单页面为准。
```

## 注意事项

- 不要在服务器配置 GoDaddy、Namecheap、Dynadot 等商业 API Key。
- 确保服务器允许出站 HTTPS 443 和 TCP 43。
- 并发不要开太夸张，建议从 20-50 起步，避免被公共数据源限速。
- 若客户量很大，建议后续加缓存：同一个域名 24 小时内不要重复查公共源。
