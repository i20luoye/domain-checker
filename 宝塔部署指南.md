# 宝塔面板部署指南

> 适用系统：CentOS / Ubuntu / Debian（装了宝塔面板的服务器）

---

## 一、核心思路

```
用户浏览器 → 你的域名（已备案） → 宝塔 Nginx（80/443）
                                  → 反向代理到 Python 后端（127.0.0.1:8000）
                                  → RDAP 数据源 / CNNIC whois
```

宝塔充当了 3 个角色：
1. **Nginx 反向代理**：接收外部请求，转发给 Python 后端
2. **SSL 证书管理**：一键申请 Let's Encrypt 证书
3. **进程守护**：Python 后端挂了自动重启

---

## 二、服务器要求

| 项目 | 要求 |
|---|---|
| 操作系统 | Linux（宝塔面板已安装） |
| 软件 | 宝塔面板 8.x+ |
| Python | 3.8+（宝塔应用商店可装） |
| 出站端口 | 443（HTTPS 访问 RDAP 源） |
| 出站端口 | 43（TCP，查询 .cn whois） |

## 三、部署步骤（10 分钟）

### Step 1：准备代码

**方式 A（推荐）：从 GitHub 拉取**

进入宝塔「终端」或 SSH 连接服务器：

```bash
cd /www/wwwroot/
git clone https://github.com/i20luoye/domain-checker.git domain-checker
```

> 如果服务器没装 git：`yum install git -y`（CentOS）或 `apt install git -y`（Ubuntu）

**方式 B：手动上传**

在本地把代码压缩成 zip，通过宝塔「文件」管理上传到 `/www/wwwroot/domain-checker/`，然后解压。

---

### Step 2：安装 Python 依赖

在宝塔「终端」执行：

```bash
cd /www/wwwroot/domain-checker
pip3 install aiohttp
```

如果提示 `pip3: command not found`，先安装 Python 3：

```
# 在宝塔应用商店 → 搜索"Python项目管理器" → 安装
# 或者通过终端安装：
apt install python3-pip -y   # Ubuntu
yum install python3-pip -y   # CentOS
```

---

### Step 3：配置 Python 项目（两种方法选一种）

#### 方法 A：使用宝塔「Python项目管理器」（推荐）

1. 宝塔面板 → **应用商店** → 搜索 **Python项目管理器** → 安装
2. 打开 Python项目管理器 → 点击 **添加项目**
3. 填写：

| 项目 | 值 |
|---|---|
| 项目名称 | `domain-checker` |
| Python版本 | 选择 3.8+ |
| 框架 | `Flask`（虽不是 Flask，但选这个不影响，主要是用它管理进程） |
| 项目路径 | `/www/wwwroot/domain-checker` |
| 启动文件 | `domain_server.py` |
| 监听端口 | `8000` |
| 启动方式 | `python` |

4. 点 **确定** → 宝塔会自动创建虚拟环境、安装依赖、启动服务
5. 检查状态：**运行中**（绿色）

> ⚠️ **常见问题**：如果启动报错，检查 Python 项目设置里的「启动文件」和「项目路径」是否正确。

#### 方法 B：手动启动（适合熟悉命令行的）

```bash
cd /www/wwwroot/domain-checker
nohup python3 domain_server.py --host 127.0.0.1 --port 8000 > server.log 2>&1 &
```

---

### Step 4：配置 Nginx 反向代理

1. 宝塔 → **网站** → **添加站点**
2. 填写你的域名（如 `check.example.com`）
3. 创建后进入站点设置

**关键：设置反向代理**

网站设置 → **反向代理** → **添加反向代理**：

| 项目 | 值 |
|---|---|
| 代理名称 | `domain-checker` |
| 目标 URL | `http://127.0.0.1:8000` |
| 发送域名 | `$host` |

宝塔会自动生成 Nginx 配置，类似这样：

```nginx
location / {
    proxy_pass http://127.0.0.1:8000;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;

    # WebSocket 支持（如果需要）
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";

    # 取消缓存限制（避免大请求被截断）
    proxy_buffering off;
    proxy_request_buffering off;
    client_max_body_size 10m;
}
```

---

### Step 5：申请 SSL 证书

宝塔 → **网站** → 你的站点 → **SSL** → **Let's Encrypt** → 勾选域名 → 申请

---

### Step 6：验证

浏览器访问 `https://check.example.com/api/health`

返回类似：
```json
{
    "ok": true,
    "source": "rdap",
    "commercialApiEnabled": false,
    "supportedSuffixes": ["ai", "app", "cn", "com", "dev", "io", "net", "org"]
}
```

然后打开 `https://check.example.com/` 就能看到前端页面。

## 四、端口说明

| 端口 | 方向 | 用途 |
|---|---|---|
| 80 | 入站 | HTTP（Nginx 自动跳转 HTTPS） |
| 443 | 入站 | HTTPS |
| 8000 | 本机 | Python 后端（Nginx 转发到这，外部不可直接访问） |
| 443 | 出站 | 查 RDAP 数据源（HTTPS） |
| 43 | 出站 | 查 .cn 域名 whois（TCP） |

**安全注意**：8000 端口只在 `127.0.0.1` 监听，不需要在宝塔「安全」里放行。如果 Nginx 设置正确，外部扫不到 8000 端口。

## 五、常见问题

### Q1：查 .cn 域名全部失败

检查服务器能否出站 TCP 43：
```bash
# 宝塔安全 → 防火墙 → 看看有没有禁止出站 43 端口
# 或直接测试连通性：
timeout 5 bash -c 'echo "google.cn" | nc whois.cnnic.cn 43' && echo "OK"
```

### Q2：500 错误 / 502 Bad Gateway

通常是 Python 服务挂了：
1. 宝塔 → Python项目管理器 → 查看「日志」
2. 常见原因：aiohttp 没装，或代码路径不对

### Q3：查了大量域名后变慢

免费数据源的限流保护。可以加缓存或降低并发（默认 30，降到 15）。

### Q4：域名查询结果全是"可注册"

可能网络环境对 RDAP 服务器出站被拦截。测试：
```bash
curl -s -o /dev/null -w "%{http_code}" https://rdap.verisign.com/com/v1/domain/google.com
```
返回 200 表示正常。

```bash
# 查看实时日志
tail -f /www/wwwroot/domain-checker/server.log

# 重启服务（Python项目管理器 → 重启）

ps aux | grep domain_server  # 找到 PID

# 更新代码
cd /www/wwwroot/domain-checker
git pull
# 重启 Python 项目管理器中的项目
```

## 七、可选：开启缓存提升速度

在服务器上建一个 SQLite 缓存文件：

```bash
touch /www/wwwroot/domain-checker/cache.db
chmod 666 /www/wwwroot/domain-checker/cache.db
```

然后在 `domain_server.py` 同目录下放一个 `config.json`：

```json
{
    "cache_db": "cache.db",
    "cache_ttl_hours": 24,
    "max_workers": 30
}
```

查询过的域名 24 小时内秒回，不会重复请求 RDAP。
