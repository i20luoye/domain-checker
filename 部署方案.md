## 一、DNS 预筛漏斗（借鉴 sithulaka）

### 为什么需要

| 当前做法 | 问题 |
|---|---|
| 每个域名 → HTTP RDAP 请求 | 几万个域名 = 几万次 HTTP 请求，慢 + 容易被限速 |
| .cn 域名 → TCP whois 连接 | 每个连接都要完整握手，不能复用 |
| **大部分域名已是已注册** | 实际能注册的不到 1%，99% 的请求浪费了 |

### 改进方案

```
输入域名 → DNS 预筛 → 有 DNS 记录？ → ✅ 标记"已注册"（5ms）
                     → 无 DNS 记录？ → RDAP/WHOIS 确认（500ms）
```

```python
import socket

def quick_dns_check(domain: str) -> bool:
    """
    DNS 预筛：检查域名是否有任何 DNS 记录。
    返回 True  = 有 DNS 记录 → 99% 是已注册域名
    返回 False = 无 DNS 记录 → 需要 RDAP/WHOIS 进一步证实
    """
    try:
        # 查 A 记录（IPv4）
        socket.getaddrinfo(domain, 80, socket.AF_INET, socket.SOCK_STREAM)
        return True
    except socket.gaierror:
        try:
            # 查 AAAA 记录（IPv6）
            socket.getaddrinfo(domain, 80, socket.AF_INET6, socket.SOCK_STREAM)
            return True
        except socket.gaierror:
            try:
                # 查 MX 记录（邮件服务器）
                import dns.resolver  # dnspython 库
                dns.resolver.resolve(domain, 'MX')
                return True
            except Exception:
                return False
```

### 效果预估

| 指标 | 当前 | 加 DNS 预筛后 |
|---|---|---|
| 已注册域名判定速度 | 500ms-2s（HTTP RDAP） | **5ms**（本地 DNS 查询） |
| 对 RDAP 服务器的请求量 | 100%（每个域名都请求） | **~20%**（只有无 DNS 的才请求） |
| 被限速概率 | 高（大量重复请求） | **低**（请求量减少 80%） |
| 总查询时间（17,576 个） | ~2-3 分钟 | **~30-60 秒** |

## 二、IANA 动态路由（借鉴 saidutt46）

### 为什么需要

当前代码硬编码了 RDAP_SOURCES 字典，只支持 8 种后缀：

```python
RDAP_SOURCES = {
    "com": ["https://rdap.verisign.com/..."],
    "cn":  ["whois://whois.cnnic.cn:43"],
    # 只有 com/net/org/cn/io/ai/app/dev 8 种
}
```

全球实际有 **1200+ TLD**。用户想查 .xyz/.top/.shop/.icu 等新后缀就会报错。

### 改进方案

在程序启动时拉取 IANA 官方 Bootstrap 文件，自动发现所有 TLD 的 RDAP 服务器。

```python
import json, aiohttp

IANA_BOOTSTRAP_URL = "https://data.iana.org/rdap/dns.json"
# 本地缓存文件
IANA_CACHE_FILE = "iana_bootstrap.json"

async def load_iana_bootstrap() -> dict:
    """加载 IANA RDAP Bootstrap，返回 {tld: [rdap_urls]}"""
    # 优先从本地缓存加载（减少启动时间）
    import os
    if os.path.exists(IANA_CACHE_FILE):
        with open(IANA_CACHE_FILE) as f:
            data = json.load(f)
            return _parse_bootstrap(data)

    # 从 IANA 官方拉取
    async with aiohttp.ClientSession() as session:
        async with session.get(IANA_BOOTSTRAP_URL) as resp:
            data = await resp.json()

    # 缓存到本地
    with open(IANA_CACHE_FILE, "w") as f:
        json.dump(data, f)

    return _parse_bootstrap(data)

def _parse_bootstrap(data: dict) -> dict:
    """解析 IANA Bootstrap 格式为 {tld: [urls]}"""
    result = {}
    for entry in data.get("services", []):
        tlds = entry[0]        # ["com", "net"]
        servers = entry[1]     # [{"top-most": "...", ...}]
        urls = [s["top-most"] for s in servers if "top-most" in s]
        for tld in tlds:
            tld_clean = tld.lstrip(".")
            result[tld_clean] = urls
    return result
```

### 效果

| 当前 | 改进后 |
|---|---|
| 只支持 8 种后缀 | ✅ **自动支持 1200+ TLD** |
| 新增后缀要改代码 | ✅ 重启即生效，无需改代码 |
| 没有后备源 | ✅ 自动获取权威 RDAP 列表 |

### 对 .cn 的特殊处理

IANA Bootstrap 中 .cn 的 RDAP 路由到 iana.org（返回 404），而 CNNIC 实际可用的是 whois 端口 43。所以我们需要在 IANA 数据之上加一层**手动覆盖规则**：

```python
# 已知的问题 TLD 覆盖（IANA 路由不准确）
TLD_OVERRIDES = {
    "cn": ["whois://whois.cnnic.cn:43"],  # CNNIC whois
}
```

## 三、指数退避重试（借鉴 beast-domain-checker）

### 为什么需要

当前代码遇到网络错误或限速时直接报错，用户需要手动点「重查失败」。对大批量扫描来说不友好。

### 改进方案

```python
import asyncio

async def query_with_retry(session, url, max_retries=3):
    """带指数退避的重试查询"""
    for attempt in range(1, max_retries + 1):
        try:
            async with session.get(url, timeout=aiohttp.ClientTimeout(total=8)) as resp:
                if resp.status == 429:
                    # 被限速了！等待后重试
                    retry_after = int(resp.headers.get("Retry-After", 2 ** attempt))
                    wait = min(retry_after, 2 ** attempt * 1.5)  # 指数退避
                    await asyncio.sleep(wait)
                    continue
                return {"ok": True, "status": resp.status}
            except (asyncio.TimeoutError, aiohttp.ClientError):
                if attempt < max_retries:
                    wait = 1.5 ** attempt  # 1.5s → 2.25s → 3.375s
                    await asyncio.sleep(wait)
                    continue
                return {"ok": False, "error": "max_retries_exceeded"}
```

### 重试策略表

| 失败原因 | 第 1 次重试 | 第 2 次重试 | 第 3 次重试 |
|---|---|---|---|
| 网络超时 | 等 1.5s | 等 2.25s | 等 3.4s → 失败 |
| HTTP 429（限速） | 按 Retry-After 头 或 2s | 4s | 8s → 失败 |
| HTTP 503（服务不可用） | 等 1.5s | 等 3s | 等 6s → 失败 |
| DNS 解析失败 | 等 1s | 等 2s | 等 4s → 失败 |

## 四、Provider 抽象层（架构重构）

### 当前问题

代码中 RDAP、whois、GoDaddy 的查询逻辑散落在不同函数里，没有统一的接口。

### 改进方案

```python
from abc import ABC, abstractmethod

class DomainProvider(ABC):
    """域名查询提供者接口"""
    @abstractmethod
    async def check(self, domain: str, suffix: str) -> dict:
        """
        返回:
        {
            "status": "available" | "taken" | "error",
            "price_usd": float | None,
            "currency": str | None,
            "premium": bool,
            "method": str,
        }
        """
        pass

class RDAPProvider(DomainProvider):
    """RDAP 多源投票提供者"""
    async def check(self, domain: str, suffix: str) -> dict:
        # RDAP 查询逻辑...

class WhoisProvider(DomainProvider):
    """WHOIS 端口 43 提供者"""
    async def check(self, domain: str, suffix: str) -> dict:
        # WHOIS 查询逻辑...

class NamecheapProvider(DomainProvider):
    """Namecheap API 提供者（可选配置）"""
    async def check(self, domain: str, suffix: str) -> dict:
        # Namecheap API 查询逻辑...

class GoDaddyProvider(DomainProvider):
    """GoDaddy API 提供者（可选配置）"""
    async def check(self, domain: str, suffix: str) -> dict:
        # GoDaddy 批量查询逻辑...

class CheckEngine:
    """查询引擎：组合 DNS 预筛 + Provider 自动降级"""
    def __init__(self):
        self.providers = {
            "rdap": RDAPProvider(),
            "whois": WhoisProvider(),
        }
        # 如果配置了 API Key，注册商业提供者
        if os.environ.get("NAMECHEAP_API_KEY"):
            self.providers["namecheap"] = NamecheapProvider()
        if os.environ.get("GODADDY_KEY"):
            self.providers["godaddy"] = GoDaddyProvider()

    async def check(self, domain: str, suffix: str) -> dict:
        # 第 1 步：DNS 预筛
        if quick_dns_check(domain):
            return {"status": "taken", "method": "dns"}

        # 第 2 步：按优先级使用 Provider
        for name, provider in self.providers.items():
            result = await provider.check(domain, suffix)
            if result["status"] != "error":
                return result

        return {"status": "error", "method": "all_failed"}
```

### Provider 优先级链

```
DNS 预筛（5ms）→ RDAP 多源投票（首选，免费）
                → WHOIS 端口 43（备选，免费，用于古老 TLD）
                → Namecheap API（如果配置了 Key，可检测溢价）
                → GoDaddy API（如果配置了 Key，可检测溢价）
```

## 八、深度调研中发现的额外关键设计（也值得加）

以下是从三个项目的源码中挖掘到的、方案 v3 未完整覆盖的设计点：

### 8.1 三重 IANA 缓存（借鉴 saidutt46）

当前方案只设计了「本地文件缓存」，可以升级为三层：

```
第 1 层：硬编码 32 个常用 TLD（com/cn/io/ai 等）
   → 零等待，秒回

第 2 层：进程级内存缓存（OnceLock + 24h TTL）
   → 启动后从 IANA 拉取，存在内存
   → 24 小时后自动刷新

第 3 层：持久化文件缓存（iana_bootstrap.json）
   → 进程重启时先读本地文件
   → 减少网络请求
```

此外加 **负缓存**（no_rdap 集合）：记录哪些 TLD 没有 RDAP 服务，避免反复尝试失败。

### 8.2 WHOIS 服务器动态发现（借鉴 saidutt46）

当前 .cn 的 whois 服务器是硬编码的 `whois.cnnic.cn:43`。更通用的做法：

```python
def discover_whois_server(tld: str) -> str | None:
    """通过 IANA 查询指定 TLD 的权威 WHOIS 服务器"""
    import subprocess
    result = subprocess.run(
        ["whois", "-h", "whois.iana.org", tld],
        capture_output=True, text=True, timeout=10
    )
    for line in result.stdout.splitlines():
        if line.strip().startswith("refer:"):
            return line.split(":", 1)[1].strip()
    return None  # 回退到已知默认服务器
```

**加上三层缓存**：
- 硬编码已知的 whois 服务器（cnnic、verisign 等）
- 进程缓存（避免同一 TLD 反复发现）
- IANA 远程发现

### 8.3 置信度评分系统（借鉴 sithulaka）

不是简单的二元判定，而是给每个结果一个置信度等级：

| 置信度 | 判定条件 | 用户看到 |
|---|---|---|
| **VERY HIGH** | RDAP 404 或 DNS 有记录 | ✅ 可注册 / ❌ 已注册 |
| **HIGH** | RDAP 200（已注册） | ❌ 已注册 |
| **MEDIUM** | RDAP 失败 + WHOIS 返回可用 | ⚠️ 可能可注册 |
| **LOW** | 所有源都失败，仅 DNS 无记录 | 💡 建议手动确认 |

这样用户在查询大量域名时，能根据置信度决定优先级。

### 8.4 三元结果 + 语义化错误（借鉴 saidutt46）

```python
from enum import Enum
from dataclasses import dataclass, field
from typing import Optional

class Availability(Enum):
    AVAILABLE = "available"
    TAKEN = "taken"
    UNKNOWN = "unknown"  # 无法确定

@dataclass
class CheckResult:
    domain: str
    availability: Availability
    confidence: str = "LOW"       # VERY HIGH / HIGH / MEDIUM / LOW
    premium: bool = False
    price_usd: Optional[float] = None
    method: str = "unknown"       # dns / rdap / whois / consensus
    error: Optional[str] = None

    def indicates_available(self) -> bool:
        """语义化错误判断：某些错误暗示域名可能是可注册的"""
        if self.availability == Availability.AVAILABLE:
            return True
        if self.error and "404" in self.error:
            return True  # RDAP 404 = 域名不存在
        return False
```
